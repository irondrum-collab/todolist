## 1. 제품 개요

### 목적
인증된 사용자가 카테고리와 날짜 기반으로 할 일을 등록·관리하고, 상태별 필터링으로 진행 현황을 파악할 수 있는 웹 애플리케이션

### 목표 사용자
학생 및 20~50대 직장인

### 비즈니스 목표
- 개인 할 일 관리의 디지털화
- 카테고리·상태별 분류로 업무 우선순위 파악 지원

## 2. 기능 요구사항

### UC-01: 회원가입

**정상 흐름**
1. 사용자가 이메일, 비밀번호, 이름을 입력 후 제출
2. 유효성 검사 통과 시 계정 생성
3. 로그인 페이지로 이동

**입력 유효성**
- 이메일: RFC 5322 형식, 최대 254자, 중복 불가
- 비밀번호: 8~128자, 영문·숫자·특수문자 각 1자 이상
- 이름: 1~50자

**예외 처리**
- 이메일 중복: "이미 사용 중인 이메일입니다"
- 형식 오류: 필드별 인라인 오류 메시지 표시
- 필수 항목 누락: 제출 차단

---

### UC-02: 로그인

**정상 흐름**
1. 이메일·비밀번호 입력 후 제출
2. 인증 성공 시 JWT 발급, 할 일 목록 페이지로 이동

**예외 처리**
- 이메일 미존재 또는 비밀번호 불일치: "이메일 또는 비밀번호가 올바르지 않습니다"
- 인증 토큰 만료: 로그인 페이지로 리다이렉트

---

### UC-03: 내 정보 수정

**정상 흐름 (v1.0)**
1. 현재 이름·비밀번호 표시
2. 항목 수정 후 저장
3. 성공 메시지 표시

**정상 흐름 추가 (v2.0)**
- 테마·언어 설정 항목 표시 및 수정 가능
- 변경된 테마·언어 즉시 UI 적용 및 DB 저장

**제약**
- 본인 정보만 수정 가능 (DR-USER-01)
- 비밀번호 변경 시 현재 비밀번호 확인 필요
- `v2.0` 테마·언어 변경은 DB에 저장, 다음 로그인 시 유지 (DR-USER-02)

**예외 처리**
- 현재 비밀번호 불일치: "현재 비밀번호가 올바르지 않습니다"
- 입력 유효성 오류: 필드별 인라인 메시지

---

### UC-04: 할 일 등록

**정상 흐름**
1. 제목 입력 (필수)
2. 설명 입력 (선택, 최대 1,000자)
3. 카테고리 선택 (미선택 시 '기본' 자동 적용)
4. 시작일자·종료일자 캘린더 선택 (선택)
5. 저장 시 목록에 추가

**제약**
- 제목: 1~100자 (DR-TODO-01)
- 날짜: 캘린더 UI로만 선택 (DR-TODO-02)
- 종료일자 >= 시작일자 (DR-TODO-03)
- 날짜 미지정 할 일은 항상 '진행 중' 상태

**예외 처리**
- 제목 미입력: 제출 차단, "제목을 입력해 주세요"
- 종료일자 < 시작일자: "종료일자는 시작일자 이후여야 합니다"
- 미인증 요청: 401 응답, 로그인 페이지 이동

---

### UC-05: 할 일 목록 조회 (필터링)

**정상 흐름**
1. 로그인 사용자의 할 일 목록 표시
2. 카테고리 필터, 상태 필터 적용 가능
3. 각 항목에 제목·상태·카테고리·날짜 표시

**상태 계산 규칙** (서버 또는 클라이언트에서 오늘 날짜 기준 계산)

| 상태 | 조건 |
|------|------|
| 시작 전 | 완료=false, 오늘 < 시작일자 |
| 진행 중 | 완료=false, 시작일자 <= 오늘 <= 종료일자 |
| 완료 | 완료=true (날짜 무관) |
| 기한 초과 | 완료=false, 오늘 > 종료일자 |
| 진행 중 (날짜 없음) | 완료=false, 시작일자·종료일자 모두 없음 |

**필터 옵션**
- 카테고리: 전체 / 개별 카테고리
- 상태: 전체 / 시작 전 / 진행 중 / 완료 / 기한 초과

**예외 처리**
- 미인증 요청: 401 응답, 로그인 페이지 이동
- 할 일 없음: "등록된 할 일이 없습니다" 안내 문구 표시

---

### UC-06: 할 일 수정

**정상 흐름**
1. 기존 데이터 폼에 미리 채워진 상태로 표시
2. 제목·설명·카테고리·날짜·완료 여부 수정
3. 저장 시 목록 갱신

**제약**
- 등록한 사용자만 수정 가능 (DR-TODO-04)
- 제목, 날짜, 종료일자 유효성은 등록과 동일

**예외 처리**
- 타인의 할 일 수정 시도: 403 응답
- 미인증 요청: 401 응답

---

### UC-07: 할 일 삭제

**정상 흐름**
1. 삭제 버튼 클릭 → 확인 다이얼로그 표시
2. 확인 시 삭제 처리, 목록에서 제거

**제약**
- 등록한 사용자만 삭제 가능 (DR-TODO-04)

**예외 처리**
- 타인의 할 일 삭제 시도: 403 응답
- 미인증 요청: 401 응답

---

### UC-10: 카테고리 CRUD

**정상 흐름**
1. 카테고리 목록 조회: 본인 소유 카테고리 전체 반환 (기본 카테고리 포함)
2. 카테고리 등록: 이름(1~30자) 입력 후 생성
3. 카테고리 수정: 이름 변경 저장
4. 카테고리 삭제: 삭제 처리, 소속 할 일의 카테고리는 연쇄 삭제

**제약**
- 본인 소유 카테고리만 수정·삭제 가능 (DR-CAT-04)
- '기본' 카테고리는 수정·삭제 불가 (DR-CAT-02, DR-CAT-03)
- 카테고리 이름: 1~30자

**예외 처리**
- '기본' 카테고리 수정·삭제 시도: 400 응답
- 타인 카테고리 수정·삭제 시도: 403 응답
- 미인증 요청: 401 응답

> **참고**: 카테고리 CRUD 백엔드 API (`/api/categories`)는 v1.0에서 구현 완료.  
> 카테고리 관리 **UI**는 프론트엔드 추후 개발 예정 (현재 '기본' 카테고리만 화면 노출).

---

### UC-08: 테마 변경 (Dark / Light Mode) `v2.0 신규`

**정상 흐름**
1. 헤더 또는 내 정보 화면에서 테마 토글 선택
2. 즉시 전체 UI에 선택한 테마 적용
3. 선택값(`dark` / `light`)을 사용자 DB에 저장
4. 다음 로그인 시 저장된 테마로 자동 적용

**제약**
- 지원 값: `light` (기본값), `dark`
- 미로그인 상태: 브라우저 로컬스토리지에 임시 저장, 로그인 후 DB 값으로 덮어씀
- DB 저장은 UC-03 내 정보 수정 API를 통해 처리

**예외 처리**
- 저장 실패 시: UI 적용은 유지, 토스트 알림으로 저장 실패 안내

---

### UC-09: 언어 변경 (다국어) `v2.0 신규`

**정상 흐름**
1. 헤더 또는 내 정보 화면에서 언어 선택
2. 즉시 전체 UI 텍스트를 선택한 언어로 전환
3. 선택값을 사용자 DB에 저장
4. 다음 로그인 시 저장된 언어로 자동 적용

**지원 언어 (MVP)**
- 한국어 (`ko`, 기본값)
- 영어 (`en`)

**제약**
- 다국어 처리는 프론트엔드에서 i18n 라이브러리로 처리
- DB에는 언어 코드(`ko`, `en`)만 저장
- 미로그인 상태: 브라우저 로컬스토리지에 임시 저장

**예외 처리**
- 지원하지 않는 언어 코드 수신 시: 기본값 `ko` 적용
- 저장 실패 시: UI 적용은 유지, 토스트 알림으로 저장 실패 안내

## 3. 비기능 요구사항

### 성능
- 최대 동시 접속자 1,000명 처리
- API 응답 시간 목표: 일반 요청 500ms 이내

### 보안
- 비밀번호: bcrypt 해싱 저장
- 인증: JWT (Access Token)
- API 엔드포인트: 인증 미들웨어로 보호 (DR-AUTH-02)
- SQL Injection 방지: pg 라이브러리 파라미터 바인딩 사용

### 반응형 UI
- 웹 우선 설계, 모바일은 반응형 웹 UI 적용 (별도 앱 없음)
- 최소 지원 뷰포트: 375px (모바일) ~ 1440px (데스크톱)

## 5. 제외 범위 (Out of Scope)

- 소셜 로그인 (Google, Kakao 등)
- 이메일 인증 및 비밀번호 찾기
- 알림 기능 (푸시, 이메일 등)
- 할 일 공유 및 협업 기능
- 카테고리 관리 UI (카테고리 CRUD 백엔드 API는 구현됨. 프론트엔드 UI는 미구현 — MVP에서는 '기본' 카테고리만 화면 노출)
- 접근성(Accessibility) 대응
- 모바일 네이티브 앱

## 6. MVP 범위 및 일정

### v1.0 — 핵심 기능 (2일, 1인 개발)

| 우선순위 | 기능 | 유스케이스 |
|---------|------|-----------|
| P0 (필수) | 회원가입, 로그인 | UC-01, UC-02 |
| P0 (필수) | 할 일 CRUD | UC-04, UC-05, UC-06, UC-07 |
| P1 (권장) | 내 정보 수정 (이름·비밀번호) | UC-03 |

**Day 1**: 백엔드 API 전체 구현 (인증 + 할 일 CRUD)  
**Day 2**: 프론트엔드 UI 구현 + 연동 + 반응형 적용

### v2.0 — 확장 기능 (v1.0 완료 후 착수)

| 우선순위 | 기능 | 유스케이스 |
|---------|------|-----------|
| P0 (필수) | 테마 변경 (Dark/Light) | UC-08 |
| P0 (필수) | 다국어 전환 (한국어/영어) | UC-09 |
| P1 (권장) | 내 정보 수정에 테마·언어 항목 추가 | UC-03 확장 |

**Day 1**: 백엔드 — User 테이블에 `theme`, `language` 컬럼 추가 (마이그레이션), 내 정보 수정 API 확장  
**Day 2**: 프론트엔드 — i18n 라이브러리 적용, 테마 토글 UI 구현, 로그인 시 설정 자동 복원
