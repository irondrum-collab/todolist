## 1. 프로젝트 개요

### 목적

인증된 사용자 개인별로 할 일을 체계적으로 관리할 수 있는 웹 애플리케이션을 제공한다.  
카테고리와 날짜(시작일/종료일) 기반으로 할 일을 등록·관리하고, 다양한 상태 필터링을 통해 진행 현황을 파악할 수 있도록 한다.

### 범위

**v1.0**
- 사용자 회원가입 및 로그인(인증)
- 사용자 개인 정보 수정 (이름, 비밀번호)
- 카테고리 기반 할 일 등록·수정·삭제·조회
- 할 일 시작일/종료일 지정
- 할 일 상태별·카테고리별 필터링 조회

**v2.0 추가**
- UI 테마 변경 (Dark / Light Mode) 및 사용자 설정 저장
- UI 다국어 지원 (한국어 / 영어) 및 사용자 설정 저장

## 2. 핵심 도메인 개념

### 사용자 (User)

인증 후 앱을 사용하는 주체.

| 속성 | 설명 | 제약 조건 |
|------|------|-----------|
| 사용자 ID | 시스템 고유 식별자 | 자동 생성, 변경 불가 |
| 이메일 | 로그인 식별자, 고유값 | RFC 5322 형식 (예: user@example.com), 최대 254자, 중복 불가 |
| 비밀번호 | 암호화 저장 | 최소 8자 이상, 영문·숫자·특수문자 각 1자 이상 포함, 최대 128자 |
| 이름 | 사용자 표시 이름 | 최소 1자, 최대 50자 |
| 테마 `v2.0` | UI 테마 설정 | `light` (기본값) / `dark` |
| 언어 `v2.0` | UI 언어 설정 | `ko` (기본값) / `en` |
| 가입일시 | 계정 생성 일시 | 자동 생성 |

### 카테고리 (Category)

할 일을 분류하는 그룹. 사용자가 직접 생성하거나 시스템 기본 카테고리를 사용한다.

| 속성 | 설명 | 제약 조건 |
|------|------|-----------|
| 카테고리 ID | 시스템 고유 식별자 | 자동 생성, 변경 불가 |
| 이름 | 카테고리 명칭 | 최소 1자, 최대 30자 |
| 소유 사용자 | 해당 카테고리를 소유한 사용자 | 필수, 사용자 ID 참조 |

### 할 일 (Todo)

사용자가 등록하는 작업 단위.

| 속성 | 설명 | 제약 조건 |
|------|------|-----------|
| 할 일 ID | 시스템 고유 식별자 | 자동 생성, 변경 불가 |
| 제목 | 할 일 내용 요약 | 필수, 최소 1자, 최대 100자 |
| 설명 | 할 일 상세 내용 (선택) | 최대 1,000자 |
| 시작일자 | 할 일을 시작하는 날짜 | 선택, YYYY-MM-DD 형식 |
| 종료일자 | 할 일을 완료해야 하는 날짜 | 선택, YYYY-MM-DD 형식, 시작일자 <= 종료일자 |
| 완료 여부 | 할 일 완료 처리 상태 (true/false) | 기본값: false |
| 카테고리 | 소속 카테고리 | 미지정 시 '기본' 카테고리 자동 적용 |
| 소유 사용자 | 할 일을 등록한 사용자 | 필수, 사용자 ID 참조 |
| 등록일시 | 할 일 생성 일시 | 자동 생성 |
| 수정일시 | 마지막 수정 일시 | 자동 갱신 |

## 3. 도메인 규칙

### 인증 규칙

| ID | 규칙 |
|----|------|
| DR-AUTH-01 | 회원가입 후 로그인해야 앱의 모든 기능을 사용할 수 있다. |
| DR-AUTH-02 | 인증되지 않은 사용자는 할 일 조회·등록·수정·삭제를 수행할 수 없다. |

### 사용자 규칙

| ID | 규칙 |
|----|------|
| DR-USER-01 | 로그인한 사용자는 자신의 정보(이름, 비밀번호)만 수정할 수 있다. |
| DR-USER-01-v2 `v2.0` | DR-USER-01 확장: 수정 가능한 정보에 테마, 언어가 추가된다. |
| DR-USER-02 `v2.0` | 테마와 언어 설정은 DB에 저장되며, 다음 로그인 시 저장된 값이 자동 적용된다. |
| DR-USER-03 `v2.0` | 미로그인 상태의 테마·언어 선택은 브라우저 로컬스토리지에 임시 저장되며, 로그인 후 DB 값으로 덮어쓴다. |

### 카테고리 규칙

| ID | 규칙 |
|----|------|
| DR-CAT-01 | 할 일 등록 시 카테고리를 지정하지 않으면 '기본' 카테고리가 자동 적용된다. |
| DR-CAT-02 | '기본' 카테고리는 시스템이 제공하며 삭제할 수 없다. |
| DR-CAT-03 | '기본' 카테고리는 이름을 수정할 수 없다. |
| DR-CAT-04 | 카테고리의 수정 및 삭제는 해당 카테고리를 소유한 사용자만 가능하다. |

### 할 일 규칙

| ID | 규칙 |
|----|------|
| DR-TODO-01 | 할 일은 반드시 제목을 포함해야 한다. |
| DR-TODO-02 | 시작일자와 종료일자는 캘린더를 통해 선택한다. |
| DR-TODO-03 | 종료일자는 시작일자와 같거나(>=) 이후여야 한다. 즉, 종료일자 >= 시작일자. |
| DR-TODO-04 | 할 일의 수정 및 삭제는 해당 할 일을 등록한 사용자만 가능하다. |

### 할 일 상태 규칙

할 일 상태는 시작일자, 종료일자, 완료 여부의 조합으로 결정된다. 모든 날짜 비교는 당일 날짜(오늘)를 기준으로 한다.

| ID | 상태 | 조건 |
|----|------|------|
| DR-STATUS-01 | 시작 전 | 완료 여부 = false 이고, 오늘 < 시작일자 (시작일자 당일 미포함) |
| DR-STATUS-02 | 진행 중 | 완료 여부 = false 이고, 시작일자 <= 오늘 <= 종료일자 (시작일자·종료일자 당일 포함) |
| DR-STATUS-03 | 완료 | 완료 여부 = true |
| DR-STATUS-04 | 기한 초과 | 완료 여부 = false 이고, 오늘 > 종료일자 (종료일자 당일 미포함) |
| DR-STATUS-05 | 진행 중 (날짜 없음) | 완료 여부 = false 이고, 시작일자·종료일자 모두 미지정 |

## 4. 유비쿼터스 언어

| 용어 | 정의 |
|------|------|
| 사용자 (User) | 회원가입하고 인증된 앱 이용자 |
| 테마 (Theme) | UI의 색상 모드. `light`(밝은 화면) 또는 `dark`(어두운 화면) |
| 언어 (Language) | UI 표시 언어. `ko`(한국어) 또는 `en`(영어) |
| 인증 (Authentication) | 이메일과 비밀번호로 사용자 신원을 확인하는 행위 |
| 할 일 (Todo) | 사용자가 수행해야 할 작업 단위 |
| 카테고리 (Category) | 할 일을 분류하는 그룹 |
| 기본 카테고리 | 카테고리 미지정 시 자동 적용되는 시스템 기본 분류 |
| 시작일자 | 할 일을 시작하는 날짜 |
| 종료일자 | 할 일을 완료해야 하는 목표 날짜 |
| 완료 | 할 일이 처리 완료된 상태 |
| 기한 초과 | 종료일자 다음날부터 완료되지 않은 상태 (오늘 > 종료일자) |
| 진행 중 | 시작일자 당일 이상(>=), 종료일자 당일 이하(<=)이며 완료되지 않은 상태 |
| 시작 전 | 오늘이 시작일자 미만(<)인 상태 |
| 진행 중 (날짜 없음) | 시작일자·종료일자 모두 미지정이며 완료되지 않은 상태 |
